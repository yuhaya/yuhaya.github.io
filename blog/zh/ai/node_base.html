基礎節點說明 ​

自定義設計AI工作流的時候，基礎節點都會用到，基礎節點大致知道如何使用後，其它型別節點也大致原理相同，下面是一些主要的基礎節點的功能說明，和應用場景

啟動入口 ​

功能說明 ​

每一個工作流都有且只有一個啟動入口節點，當我們啟動工作流的時候，系統會第一個尋找到啟動入口節點，然後從啟動入口節點開始執行工作流

配置說明 ​

可以建立多個輸出埠，每個輸出埠可以設定一些預設輸出資料，如果輸出埠設定的有預設值，工作流啟動後會自動將值透過連線傳遞給下一個節點，如果沒有預設值，並且此埠的允許為空開關開啟時，會傳遞一個空字串給下一個節點。

應用場景 ​

充當每一個工作流的開始入口，假設此節點的輸出埠預設值設定10個TikTok博主主頁地址，他的下一個節點是TikTok粉絲採集，那麼工作流啟動後會自動將10個TikTok博主主頁地址傳遞給TikTok粉絲採集節點，執行粉絲自動化採集。

資料分身 ​

功能說明 ​

資料分身節點可以建立多個輸出埠，其實就是將輸入埠的資料複製到每一個輸出埠，每個輸出埠連線下一個節點執行下一步資料處理，實現任務的並行處理。

應用場景 ​

上面工作流有兩個資料分身節點，我們來分別分析前後兩個資料分身節點的作用

  • 第一個資料分身節點

用於連線啟動入口節點和粉絲採集節點，將啟動入口節點輸出的資料複製到兩個輸出埠，一個連線粉絲採集節點的任務驅動型埠，一個連線粉絲採集節點的關鍵資料型埠。為什麼不能直接啟動入口節點輸出埠連線到粉絲採集節點的任務驅動型埠呢？

按照上期 執行流程 的教程說明，任務驅動型埠有資料進入，才能觸發節點執行任務，但是如果節點存在關鍵資料型埠，必須關鍵資料型埠有資料進入，才能真正的根據關鍵資料型埠資料量建立執行任務，否則並不能真實建立任務並執行，會直接視作沒有任務需要執行從而結束。

如果直接啟動入口節點輸出埠連線到粉絲採集節點的任務驅動型埠，那麼當啟動入口節點輸出埠有資料進入時，粉絲採集節點會執行任務，但是粉絲採集節點的關鍵資料型埠沒有資料進入，這時節點想要建立採集任務，發現關鍵資料型埠沒有可供建立執行的資料，那麼會直接結束任務。

所以針對那種既有任務驅動型埠又有關鍵資料型埠的情況，一般我們使用資料分身節點來連線，可以保證兩個埠都有資料進入時，節點才能執行任務。

  • 第二個資料分身節點

第二個資料分身節點後面連線了兩個節點，一個連線批次發私信節點節點，一個連線AI使用者特徵識別節點，這樣這樣就可以同時執行兩類任務。

資料分身節點的輸出埠可以自由的增加，按照之前講解的 執行流程 裡面的埠型別說明，分別引出兩條線連線後續節點的任務驅動型埠和關鍵資料型埠。

知識檢索 ​

功能說明 ​

此節點一個輸入接收問題內容，節點內部查詢對應本地知識庫，利用AI分析推理能力，綜合問題以及查詢的相關內容，給出答案，答案透過輸出埠傳遞到下一個節點。

應用場景 ​

比如我們要構建AI客服的時候，客戶諮詢產品引數相關問題，我們可以使用知識檢索節點來自動檢索產品相關資訊後自動回答客戶問題。

生成回答 ​

功能說明 ​

此節點和知識檢索節點類似，區別在於生成回答節點直接是根據問題生成回答，不會查詢本地知識庫。

應用場景 ​

比如當我們採集到社媒上的評論內容了，需要透過評論內容來判定這個使用者的是否是比較有意向的潛在客戶，那麼我們就可以使用生成回答節點，設定合理的系統提示詞來根據評論內容來自動判定這個使用者的級別，不同級別輸出不同的文字，後續用條件節點根據不同級別輸出不同的文字來執行不同的任務。

問題分類 ​

功能說明 ​

利用AI大模型將輸入的問題進行分類，不同分類對應不同的輸出埠，如果與該分類匹配，則將問題傳遞給該輸出埠並遞給下一個節點。

應用場景 ​

當我們做AI客服的時候，需要先根據使用者的問題，判斷是產品引數諮詢，還是售後維修諮詢，還是價格諮詢，還是其它諮詢，我們可以使用問題分類節點來將問題進行分類，AI自動分析判斷，根據分類對應的埠傳遞給下一個節點做針對性處理。

速率控制 ​

功能說明 ​

此節點接收到輸入資料之後，不會立馬傳到輸出埠，而是等待一定時間後，再傳到輸出埠將資料傳遞給下一個節點，一般用於控制後續節點執行任務的速率。

應用場景 ​

比如上一個節點是TK粉絲採集，採集到1000條TK粉絲，需要執行批次加好友節點，但是不想任務執行太密集，那麼我們就可以使用速率控制節點放在中間來控制節點執行任務的速率。

靜態訊息 ​

功能說明 ​

此節點接收到輸入資料之後，從預設的靜態文字中選擇一條文字，將文字傳遞給下一個節點，會忽略本身的輸入埠傳入的資料。

應用場景 ​

當我們用到問題分類節點，碰到某一類未知問題時，需要直接輸出一個固定文字，那麼可以將問題分類節點中的那個位置問題分類輸出埠連到此靜態訊息節點埠，增強確定性輸出。

指令碼 ​

功能說明 ​

此節點執行Dart指令碼，可以自定義指令碼程式碼，一般用於做資料轉換，資料清洗，資料處理等。

應用場景 ​

dart
String main(String p1,String p2){
  return "$p1:$p2";
}

上面程式碼是一個簡單的Dart指令碼，意思將輸入埠p1和輸入埠p2的值以:拼接起來傳遞給輸出埠。

以上只是簡單的舉例，Dart指令碼可以自由發揮，可以處理非常複雜的情況，Dart程式碼不知道如何編寫，可以利用AI工具幫忙生成。具體的書寫規範，節點的使用規則有說明。

網路請求 ​

功能說明 ​

此節點用於傳送網路請求，一般用於傳送HTTP請求，獲取網路資料，可自定義請求方式，請求頭，請求體，請求引數等。

應用場景 ​

您想接入第三方提供了API介面的服務，可以透過此方式實現

條件分支 ​

根據特定規則，匹配輸入的內容，送達到指定的輸入埠，用於連線後續不同的任務處理節點

使用提醒 ​

可以建立多個輸出埠，每個輸出埠的匹配文字可以設定多個

影片下載 ​

透過影片帖子的分享連結，下載影片到本地，將儲存的路徑透過輸出埠往後傳遞，做下一步處理。

使用提醒 ​

主流社媒都支援，Youtube，Facebook，Instagram，TikTok等全支援,需要注意一下代理設定，比如TK不對香港開放，代理不要設定香港的，不然影片下載不下來。

影片防查重 ​

自動處理影片，改變影片的各項資料，避免被平臺查重

應用場景 ​

一般可以配合影片下載節點一起使用，將下載的影片進行防查重處理，做影片搬運。

Excel儲存 ​

接收輸入的資料，以Excel檔案儲存到本地

應用場景 ​

比如上一個節點是TK粉絲採集，我們需要把採集的資訊已Excel儲存到本地，就可以把此節點加在後面自動處理。

斷開控制 ​

接收多少次輸入資料後，或者發現輸入資料包含某些文字的時候，自動斷開後續任務處理

應用場景 ​

當我們的工作流存在迴圈任務時，可以配合此節點使用，達到某些條件，自動斷開迴圈。

檔案選擇 ​

選擇一些本地檔案，將檔案路徑透過輸出埠往後傳遞，做下一步處理。

檔案上傳 ​

接收本地檔案路徑，自動上傳檔案到系統，將上傳檔案的識別符號號透過輸出節點傳遞給後續節點處理

應用場景 ​

一般像社媒自動發帖的節點，大部分可以接受已經上傳檔案的識別符號號作為附件輸入，用於發帖的時候使用，檔案上傳節點就可以把一些本地檔案上傳並轉換為的後續節點需要的識別符號號

分流器 ​

將輸入埠的資料平均分給多個輸出埠，用於連線後續不同的任務處理節點

執行結果 ​

此節點收集所有輸入的資料，當工作流執行完畢後，將所有資料彙總作為該工作流此次執行的結果。

應用場景 ​

當我們的工作流需要被其它功能呼叫的時候用到

日誌 ​

記錄顯示流經輸入埠的資料

應用場景 ​

有時後我們想檢視工作流執行過程中，流經輸入埠的資料，就可以使用此節點來記錄，插入兩個節點中間就行。
